WIP: Substructural Logics in Agda

Codebase

This was a summer research project I did with Professor Stuart Kurtz. This was my first foray into the world of programs-as-proofs and dependent type theory, and I absolutely loved it. Specifically, I formally defined several varieties of substructual logic (logics that have non-standard "structural rules", rules that say, e.g., if Z is derivable from X it is derivable from X and Y) and proved a few results both within and about those logics. It was a great way to combine my interests in CS and math, and I found the challenge of representing the organization and relationship of mathematical entities with code really satisfying. I have submitted this code for inclusion in the Agda Standard Library, and am currently in the process of some final revisions before it is officially added. I will update this entry with a link to the standard library, an updated version of just my part of the code, and a README explaining the code when I do.

Reinforcement Learning with Verified Rewards for Solving Math Problems

Report

For my "Natural Language Processing" class, I wanted to try implementing RLVR. I'm interested in how formal verification can help train LLMs to produce computer programs and mathematical proofs (and of course to verify their outputs after training). I loosely followed Allen AI's report on Tulu 3 to train a Llama 3.1 base model on solving simple math problems using open-source HuggingFace datasets, models, and algorithms (the report includes some technical discussion of the algorithms). I returned to the project after writing the above report, and was able to achieve about 0.25 accuracy after 3 epochs of training. In the original report they achieved 0.44 accuracy over 8-12 epochs so I was quite pleased with that. If compute were cheaper I would've seen what I could get for the full 12 epochs. Code is available upon request.

Training a Robot Arm with Imitation Learning

Report Presentation

I collaborated with two classmates in my "Planning, Learning and Estimation for Robotics and AI" class to train a physical robot arm to scoop out beans from a bin. Our main goal was to learn how to integrate the robotics algorithms we'd been using in class with hardware, which presented a lot of challenges. At one point I had to drive about an hour to a MicroCenter to get an extremely specific cord to get our GoPro to work as a camera for the arm. We mostly used HuggingFace's LeRobot repo with some small modifications to train our arm; using ACT imitation learning we successfully trained the arm to emulate our teleoperation to scoop beans.

GoLite Compiler in Go

I wrote this compiler for a simplified version of the language Go for my "Compilers" course. This was my first time using Go and I enjoyed the language; the project itself is by far the most complex codebase I've developed. I used ANTLR to lex and parse progams in GoLite, before constructing an Abstract Syntax Tree (AST), performing semantic analysis (e.g. ensuring all control-flow paths have a return statement), converting through multiple phases of LLVM representation, and finally producing an executable ARM assembly code file. The intricacy of a compiler even for such a simple language was amazing to me. Code and report available upon request.
